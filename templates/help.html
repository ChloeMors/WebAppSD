REQUEST: /unions

GET parameters:

state_abbr (Optional, default: '') -- return only unions located within this state
name_query (Optional, default: '') -- return only unions whose name contains this string
city (Optional, default: '') -- return only unions located within this city
members (Optional, default: '') -- return only unions whose number of members is less than this integer. If 'max' is inputted, returns unions with greater than 1,000,000 members.

REPONSE: a JSON list of dictionaries, each representing one union. Dictionaries have the following fields:

id -- (INTEGER) the ID of the union in our database
union_name -- (TEXT) the name of the union
abbr -- (TEXT) the abbreviation of the union name
unit -- (TEXT) ?
est_date -- (TEXT) date of establishment of union
members -- (TEXT) number of union members
first_name -- (TEXT) first name of union contact
last_name -- (TEXT) last name of union contact
build_num -- (TEXT) building number of union location
street_adr -- (TEXT) street address of union location
city -- (TEXT) city of union location
region -- (TEXT) state of union location
zip -- (TEXT) zip code of union location

EXAMPLE(S):

/unions/?state_abbr=AL&name_query=CARPENTERS%20IND

REQUEST: /strikes

GET parameters:

state (Optional, default: '') -- return only strikes located within this state. Use the full state name for this option.
name_query (Optional, default: '') -- return only strikes whose company contains this string
industry (Optional, default: '') -- return only strikes whose workers are within this industry
end

REPONSE:

id -- (INTEGER) the ID of the strike in our database
employer -- (TEXT) the employer of the striking workers
org -- (TEXT) the labor organization representing the workers
local -- (TEXT) local name of org
industry -- (TEXT) industry of company
BUS -- (TEXT) Bargaining unit size
locales -- (TEXT) how many locations are on strike
address -- (TEXT) location of strike
city -- (TEXT) city of strike
state -- (TEXT) state of strike
zip -- (TEXT) zip code of strike
participants -- (TEXT) approximate number of participants in strike
start_date -- (TEXT) start date of strike
end_date -- (TEXT) end date of strike
demands -- (TEXT) demands of workers on strike

EXAMPLE(S):

strikes/?state_abbr=Pennsylvania&Industry=Educational%20Services

REQUEST: /cases

GET parameters:

state_abbr (Optional, default: '') -- return only cases located within this state
name_query (Optional, default: '') -- return only cases against companies whose name includes this string
case_number (Optional, default: '') -- return the case matching this judicial case number. region (Optional, default: '') -- returns cases being tried within a particular court region, i.e. Boston, MA

REPONSE: a JSON list of dictionaries, each representing one case. Dictionaries have the following fields:

'id': row[0], 'case_name': row[1], 'case_number':row[2], 'city':row[3], 'territory':row[4], 'date_filed':row[5], 'date_closed':row[6], 'region':row[7], 'current_status':row[8], 'reason_closed':row[9], 'representative':row[10]} id -- (INTEGER) the ID of the case in our database
case_name -- (TEXT) the name of the employer against whom case is filed
case_number -- (TEXT) reference number to case's legal proceedings
city -- (TEXT) city where case is filed
territory -- (TEXT) territory (in this case, state) where case is filed
date_filed -- (TEXT) date when case was filed
date_closed -- (TEXT) date when case was closed, or ongoing
region -- (TEXT) location of court handling case
current_status -- (TEXT) refers to whether case is open or closed
reason_closed -- (TEXT) explanation for why case was closed
representative -- (TEXT) person or organization filing claim

EXAMPLE(S):

cases/?state_abbr=CA&name_query=Wellpath
[{"id": 21, "case_name": "Wellpath at San Luis Obispo County Jail", "case_number": "31-RC-286145", "city": "San Luis Obispo", "territory": "CA", "date_filed": "11/12/2021", "date_closed": "", "region": "Region 31, Los Angeles, California", "current_status": "Open", "reason_closed": "", "representative": ""}]

REQUEST: /states

No parameters. This endpoint exists for use loading dropdown menus.
Returns a list of all states/territories represented within the database.

Request: /regions

No parameters. This endpoint exists for use loading dropdown menus.
Returns a list of all judicial regions represented within the database.

REQUEST: /strike_industries

No parameters. This endpoint exists for use loading dropdown menus.
